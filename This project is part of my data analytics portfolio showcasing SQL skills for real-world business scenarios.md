📊 Pizza Sales Analysis Using SQL
📌 Project Overview

This project focuses on analyzing pizza sales data using SQL to extract meaningful business insights. The goal is to understand customer behavior, sales trends, and product performance to support data-driven decision-making.

🎯 Objectives

Analyze overall sales performance and revenue generation

Identify top-selling pizzas and high-performing categories

Understand customer ordering patterns (daily & hourly trends)

Evaluate key business metrics such as average order value

Track revenue growth over time using cumulative analysis

🛠️ Tools & Technologies

SQL (MySQL)

Relational Database Concepts

Window Functions & CTEs

Aggregations & Joins

📈 Key Insights

Identified best-selling pizzas and most popular sizes

Determined peak order hours for better resource planning

Analyzed category-wise revenue contribution

Tracked cumulative revenue trends over time

Segmented products based on demand levels

🔍 Key Features

Complex joins across multiple tables

Use of window functions (RANK, cumulative SUM)

Common Table Expressions (CTEs) for advanced analysis

KPI calculations such as revenue and average order value

Query optimization using indexing

🚀 Conclusion

This project demonstrates how SQL can be used to transform raw transactional data into actionable business insights. It highlights strong analytical thinking and the ability to solve real-world business problems using data.
